# Бэклог доработок «Научный клубок» — до идеала

> Составлен по факту ревью кода и живого прогона (2026-07). Приоритеты:
> **P0** — корректность/блокеры (жюри это увидит/сломается); **P1** — соответствие
> ТЗ и качество; **P2** — надёжность/полировка; **P3** — упаковка сдачи (формальные
> баллы); **P4** — сверх-идеал, если останется время.
> Отметка ⏳ = упирается в квоту Yandex / доступность данных.

---

## P0 — Корректность и блокеры

- [ ] **PDF-экспорт кириллицы сломан.** `klubok/export/formats.py::to_pdf` использует
  reportlab со стилями по умолчанию (Helvetica — без кириллицы) → русский текст в PDF
  пустой. Зарегистрировать TTF с кириллицей (DejaVuSans), прописать `fontName` в стилях
  Title/BodyText/Heading. Проверить на реальном ответе. *(ТЗ: экспорт PDF.)*
- [ ] **Нет временного фильтра.** `AskRequest`/`/subgraph`/`retrieve` не принимают
  диапазон дат, а ТЗ требует «материал+процесс+условия+география+**временной диапазон**»
  и вопрос «за последние 5 лет». Добавить `year_from/year_to` в API → прокинуть в
  `retrieve` → фильтр по `Publication.year`/`Experiment` → контрол в UI. Извлекать
  «за N лет» из текста вопроса (как гео/числа).
- [ ] **Верификационные поля не подтверждены.** Проверить, что извлечение реально
  заполняет `verification_level`, `geography`, `actualized_at`, `domain` на рёбрах/узлах
  (а не оставляет дефолты). Если LLM их не отдаёт — усилить промпт и/или проставлять
  эвристиками (домен по разделу корпуса, geography по языку/маркерам). *(ТЗ: модель
  верификации знаний — ключевое требование.)*
- [ ] ⏳ **Мало данных в графе (~75 узлов, 2.5 файла).** Догрузить ≥20–40 файлов,
  обязательно включить папку **«Статьи»** (авторство в имени файла → заполнит
  Expert/Facility, сейчас `/experts` пуст). Иначе демо выглядит бедно и фичи
  экспертов/дашборда/сравнения нечего показывать.
- [ ] **Надёжный запуск Neo4j.** Портативный инстанс падает между сессиями, граф не
  персистится в глазах пользователя. Сделать `scripts/start_neo4j` / `stop_neo4j`
  (или systemd/nssm-сервис), проверять доступность в `build_stores` с понятной ошибкой.
- [ ] **Stale `.lock` встроенного Qdrant.** При жёстком завершении процесса
  `data/qdrant_local/.lock` остаётся и роняет следующий старт. Авто-очистка в
  `QdrantStore` (local mode) при старте, если процесс-владелец мёртв, + graceful
  shutdown (закрывать клиент в lifespan/finally).

## P1 — Соответствие ТЗ и качество

- [ ] **Прогнать 4 эталонных вопроса ТЗ end-to-end** на наполненном графе и
  зафиксировать (скриншоты): обессоливание воды (сульфаты/хлориды 200–300 мг/л,
  сухой остаток ≤1000 мг/дм³), циркуляция католита при электроэкстракции Ni (мировая
  практика + скорость потока), Au/Ag/МПГ штейн–шлак за 5 лет, закачка шахтных вод
  РФ vs зарубеж. Доказать, что числовой/гео/временной фильтры реально сужают выдачу.
- [ ] **Мультиязычность RU/EN.** Прогнать английский вопрос (electrowinning…), проверить
  что glossary сопоставляет синонимы («электроэкстракция»/«electrowinning», «ПВП»/
  «fluidized bed furnace»). Добавить EN-пример в gold-набор. *(ТЗ: мультиязычность.)*
- [ ] **Качество извлечения.** relations F1≈0.53 на gold. Расширить `gold_set.py` до
  15–20 размеченных пассажей (устойчивая метрика), поитерировать few-shot/`_RULES`,
  сравнить yandexgpt vs yandexgpt-lite по метрикам/цене.
- [ ] **Warm-up ретривала.** Первый seed-запрос ~4.8с (холодный fulltext), потом 94мс.
  Прогревать индекс в `lifespan` API (один фиктивный `retrieve`), чтобы жюри всегда
  видело <3–5с (нефункц. требование ТЗ).
- [ ] ⏳ **Rate-limit эмбеддингов.** `YandexEmbedder` не под общим `RateLimiter` (только
  ретраи) — burst `index_chunks` может ловить квоту. Подключить лимитер/батч-паузы.
- [ ] ⏳ **Ускорить ингест под квоту.** Сейчас 0.15 rps → ~10ч на корпус. Рычаги:
  крупнее чанки (меньше LLM-вызовов), конкурентность после поднятия квоты,
  yandexgpt-lite для bulk. Главное — **поднять квоту Text Generation** в Yandex Cloud.

## P2 — Надёжность и полировка

- [ ] **Битые/сканированные PDF.** В корпусе есть сканы без текстового слоя. Проверить
  graceful-skip в `ingest_corpus` (try/except есть) + лог виновника; опционально OCR-ветка.
- [ ] **Интеграционный smoke-тест.** Сейчас 51 unit только на чистой логике. Добавить 1
  тест против поднятых Neo4j+Qdrant с MockLLM: ingest мини-документа → `/ask` → есть узлы
  и ответ. Гоняется отдельной меткой (`pytest -m integration`), не в основном прогоне.
- [ ] **Облегчить Docker-образ.** Образ тянет `sentence-transformers`→`torch` (~2–3ГБ),
  не нужный при Yandex-бэкенде. Вынести тяжёлые ML-зависимости в отдельный
  `requirements-ml.txt`, в основном Dockerfile ставить лёгкий core.
- [ ] **Логирование/мониторинг.** Единый формат логов, уровни, счётчики ошибок ингеста
  (частично в `ingest_report.json`). Опционально — /metrics.
- [ ] **Совместимость Compose.** `env_file: required:false` требует Compose v2.24+ —
  задокументировать в README или дать fallback.
- [ ] **Устойчивость к «пустому» ответу LLM** на реальных данных (частично закрыто
  `_loads_lenient`) — добить кейсы, когда модель вернула текст без JSON.

## Рекомендованный порядок под дедлайн

1. **P0-фиксы кода** (PDF-шрифт, временной фильтр, .lock, верификационные поля) — 1 день,
   это корректность и видимые жюри дефекты.
2. **Данные**: поднять квоту → догрузить «Статьи»+обзоры (параллельно с п.1).
3. **P1-проверки**: 4 вопроса ТЗ + EN + warm-up — доказательная база для видео.
4. **P3-упаковка**: репо + видео + презентация + деплой — вечер последнего дня.
5. P2/P4 — по остатку времени, не в ущерб сдаче.
