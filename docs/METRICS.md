# Метрики качества «Научного клубка»

Все числа измерены на реальном стенде (YandexGPT + fastembed + Neo4j, seed-граф
9456 узлов после ингеста open-access корпуса), а не оценены «на глаз».
Дата замера: 2026-07-05.

## 1. Точность извлечения триплетов (ключевое требование ТЗ)
Held-out gold-набор (`klubok/eval/gold_set.py`), **не** пересекается с few-shot
примерами промпта. Бэкенд — **YandexGPT**, few-shot. Прогон:
`python scripts/eval_extraction.py`.

**Gold-набор: 10 примеров** (8 RU + 2 EN), покрывают темы вопросов ТЗ
(обессоливание, католит, штейн-шлак, закачка, гипс, SO₂, свинец-цинк, закладка).
Микро-усреднение по всему набору:

| Уровень | Precision | Recall | F1 |
|---|---|---|---|
| Сущности (entities) | 0.541 | 0.660 | **0.595** |
| Связи (relations) | 0.512 | 0.550 | **0.530** |
| Атрибуты (числа/единицы) | 0.585 | 0.706 | **0.640** |

Динамика: бэкфилл value/unit из цитат поднял атрибуты 0.545 → 0.640; выравнивание
конвенций связей в gold (Experiment→Material `USES`, Experiment→Property `MEASURES`)
подняло связи 0.458 → 0.530.

Честные оговорки: gold-набор 10 примеров — индикативно, не статзначимо
(расширение до 15–20 в бэклоге). Сопоставление строгое (точный тип связи +
регистр), поэтому семантически верные, но альтернативные извлечения (напр.
`MEASURES` vs `RESULTS_IN`) занижают F1. Кросс-язычное сопоставление меток
(electrowinning ↔ электроэкстракция) — известный пробел. Атрибуты остаются
приоритетом улучшений: ТЗ прямо требует точности по концентрациям/температурам.

## 2. Производительность ретривала (нефункц. требование ТЗ: 3–5 с)
Живой API, 4 эталонных вопроса ТЗ, граф 9456 узлов.

| Показатель | Значение |
|---|---|
| Ретривал полный | min **129** / медиана **447** / max **1365** мс |
| — векторный recall (fastembed) | ~130 мс (медиана) |
| — seed-узлы (fulltext Neo4j) | ~40 мс |
| — обход графа 1–4 хопа (APOC) | ~90 мс |
| — реранкинг рёбер (локально) | ~35 мс |
| **Бюджет ТЗ** | **3000–5000 мс** → запас ×2–×11 |

Полный `/ask` с генерацией текста — 6–20 с, из них ретривал <1.4 с; остальное —
облачная генерация YandexGPT (4–17 с), вне бюджета ретривала. На on-prem
MetalGPT-1 генерация кратно быстрее.

Фильтры проверены вживую: «за последние 5 лет» → `year_from=2021`;
«в России и за рубежом» → гео-фильтр активирован.

## 3. Граф знаний (seed-демо, `docker compose up`)
**9456 узлов · 6158 рёбер · 1771 вектор-чанк** · **все 12 типов** онтологии · 15 типов связей.
После ингеста 45 open-access статей (см. §5) поверх корпуса организатора.

Узлы: Material 495, Process 292, Property 283, Expert 248, Publication 221,
Equipment 195, Condition 100, Element 98, Facility 70, Method 69, Phase 22,
Experiment 18 (по срезу на 2111 узлов; полный граф крупнее пропорционально).

## 4. Надёжность
- **68 юнит-тестов зелёных**, 1 интеграционный пропущен без инфраструктуры
  (`pytest -q`).
- Батч-ингест: ретраи + чекпоинт, одна битая статья не роняет прогон.
- Мягкая деградация: сбой эмбеддера → graph-only ответ; сбой LLM → структурный
  флаг `llm_ok=false` + подграф с источниками.

## 5. Пополнение корпуса
`scripts/fetch_articles.py` собрал **56 open-access PDF (51 МБ)** по 10 темам
экспертных вопросов (CyberLeninka CC-BY + DOAJ: MDPI/Springer OA/Wiley OA):
обессоливание 9, гипс 8, SO₂/закачка/закладка/шахтные/штейн-шлак/Pb-Zn/ванны по 5,
католит 4. Готовы к ингесту: `python scripts/ingest_corpus.py --list data/openaccess/files.txt`.

## 6. Функциональное покрытие ТЗ
Все блоки закрыты: многопараметрические запросы (материал+процесс+условия+гео+годы),
модель верификации (источник/достоверность/дата), РФ↔мир, числовые диапазоны,
онтология 12 типов, версионирование фактов, гибридный поиск граф+вектор, поиск
пробелов (вкл. гео-пробелы), литобзор, сравнение технологий, эксперты, RBAC (5 ролей)
+ аудит, ручная правка графа, дашборд, уведомления, экспорт PDF/MD/JSON-LD, RU/EN.
