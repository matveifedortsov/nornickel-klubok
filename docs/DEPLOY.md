# Развёртывание «Научного клубка»

Решение рассчитано на скромное железо: **GPU НЕ требуется** (эмбеддер — локальный
ONNX `fastembed`, генерация — по API Yandex). Укладывается в ≤16 ГБ GPU / 12 ядер /
32 ГБ RAM (реально хватает CPU + ~2–4 ГБ RAM).

## Одна команда

```bash
cp .env.example .env      # вписать YANDEX_API_KEY + YANDEX_FOLDER_ID (для связных ответов)
docker compose up -d --build
```

Поднимутся 4 сервиса:
| Сервис | Порт | Что |
|---|---|---|
| neo4j | 7474 / 7687 | граф (APOC) |
| qdrant | 6333 | векторы |
| **api** | **8000** | FastAPI (`/docs`) |
| **ui** | **8501** | Streamlit — открывать это |

**Демо-данные грузятся сами.** При первом старте API видит пустой Neo4j и
загружает `seed/` (747 узлов, 489 рёбер, 97 векторов) — граф готов к показу **без
ингеста и без Yandex-квоты** (`klubok/pipeline.py::seed_if_empty`).

## Что уже сделано для чистой сборки (важно: «ошибка сборки → 0 баллов»)
- **`.dockerignore`** исключает `.env`, корпус (гигабайты) и `runtime/` — образ
  лёгкий и без секретов.
- **Модель эмбеддера запечена в образ** (`Dockerfile`, `RUN … TextEmbedding(…)`) —
  на старте ничего не качается, работает офлайн.
- **compose форсирует** `EMBEDDER_BACKEND=fastembed` (совпадает с seed-векторами
  dim=384) и `QDRANT_MODE=server` — не зависит от наличия `.env`.
- **healthcheck** Neo4j через `cypher-shell`; api стартует только по готовности БД.
- Требует Docker Compose **v2.24+** (используется `env_file: required:false`).

## Без ключа Yandex
Сервис не падает: `get_llm` мягко откатывается в Mock. Ретривал (граф + вектор +
реранк) полностью локальный и работает; только текст ответа будет заглушкой.
С ключом (`.env`) — связные ответы YandexGPT с цитатами.

## Проверка после запуска
```bash
curl http://localhost:8000/health         # {"status":"ok","nodes":747}
# UI: http://localhost:8501  -> вкладка «Вопрос» -> кнопка примера ТЗ
```

## Публичная ссылка для жюри
Организаторы просят развёрнутое решение по ссылке. Варианты:
1. **ВМ (Yandex Cloud / любой облачный provider)**: `docker compose up -d --build`,
   пробросить порт 8501 (UI) наружу через nginx/caddy или security group.
2. Образы в внешнем реестре (Docker Hub / ghcr): собрать `docker build`, `push`,
   на ВМ `docker compose up` подтянет их.

## Пересоздать seed (если дообогатили граф)

python scripts/export_seed.py    # при поднятых Neo4j+Qdrant с данными -> seed/
